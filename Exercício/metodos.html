Métodos de Requisição

O protocolo HTTP apresenta como função traduzir o hipertexto realizando assim uma comunicação
de dados entre cliente e servidor. Para isto o HTTP utiliza métodos de requisição, que são responsáveis
para indicar o que será executado em cada ação de um determinado dado.

São 8 os métodos mais conhecidos de requisição:

MÉTODOS DE REQUISITOS FUNÇÕES
GET As requisições do método get devem retornar apenas dados.
POST Ele é utilizado frequentemente causando uma mudança no estado
do servidor, possibiitando transferência de dados.
DELET Este método é utilizado para remover um recurso específico.
PUT Este substitui represetações do recurso de destino pela carga de dados da requisição.
HEAD O head funciona de uma forma semelhante ao get,
sendo que ele não apresenta corpo.
TRACE O método trace executa um teste de chamada> com um caminho para o recurso distinto.
OPTIONS Options é utilizado para descrever as opções de comunicação com o recuro de destino.
CONECT O método conect estabelece um túnel para o servidor identificado pelo recurso de destino.
